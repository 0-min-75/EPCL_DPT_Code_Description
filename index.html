2. Main_DPT_Analysis.m

전체 분석 흐름을 관장하는 메인 스크립트

CONFIG 구성

게이트 저항값 변수 R_G를 도입하여 경로를 동적으로 생성. 예를 들어 R_G = 'Designed'이면 SiC_Slow_Designed/Raw_Data 등의 경로가 자동 조합됨.

STEP 1: Excel 조건 파일 읽기

CONFIG 구조체에 정의된 Excel 파일(예: SiC_Fast_Designed_DPT.xlsx)의 3행부터 데이터를 읽음. 열 구조는 다음과 같음.

열내용예시
AV_DC [V] — 버스 전압400
BI_DS [A] — 설정 드레인 전류200
C첫 번째 펄스 듀티—
D두 번째 펄스 듀티—
E이미지 번호—
F파형 파일 번호 (정수)3 → '00003'

F열의 정수를 sprintf('%05d')로 5자리 zero-padding하여, 하나의 동작점에 속하는 8개 파형 파일(C1~C4, Z1~Z4)의 파일명을 결정

STEP 2: 동작점 순회

각 동작점(행)에 대해 Import_Waveform_Data로 8채널 데이터를 로드하고, Extract_Switching_Metrics로 스위칭 지표를 추출한 뒤, 결과 구조체 results에 저장. 실패 시 warning을 출력하고 해당 동작점은 NaN으로 남김.

STEP 3: 결과 저장

Export_to_Simulink을 호출하여 .mat(Simulink LUT)과 .xlsx(요약 테이블)을 생성

STEP 4: 2D LUT Map 도시

Plot_LUT_Maps(results, CONFIG)을 호출하여 모든 지표의 2D 보간 맵을 생성

3. Import_Waveform_Data.m

오실로스코프 Raw Data를 파싱하고 전처리하는 모듈

3.1 파일 로딩

파일명 규칙: {채널}--Trace--{file_num_str}{확장자} (예: C1--Trace--00003.dat). 8개 채널을 순차적으로 읽음.
내부 함수 read_oscilloscope_file은 헤더 줄 수가 파일마다 달라도 자동으로 첫 번째 숫자 행을 탐지하여 (Time, Value) 2열 데이터를 추출

3.2 Deskew 보정변경 가능

전류 채널(C1/Z1, C2/Z2)의 시간축을 +10ns 이동 (오실로스코프의 Deskew 오설정을 보상하기 위함.)
→ 오설정 수정 또는 미설정 보완 용도로 사용 가능

3.3 채널 동기화

4개의 C채널과 4개의 Z채널을 각각 그룹 내에서 공통 시간 벡터로 보간함. 모든 채널의 겹치는 시간 범위를 계산한 뒤,
포인트 수가 가장 많은 채널의 시간 벡터를 기준으로 interp1(linear) 보간을 수행.

3.4 출력 구조체

wave 구조체에 t_full, I_D, V_DS, V_GS, I_out (C채널) 및 t_zoom, Z_I_D, Z_V_DS, Z_V_GS, Z_I_out (Z채널)을 담아 반환.
IGBT 모드에서는 I_C, V_CE 등의 별칭도 추가됨. meta 구조체에는 샘플링 주파수, 타임스텝, 포인트 수 등 메타 정보가 담김.

4. Extract_Switching_Metrics.m — 스위칭 지표 추출

Z채널 고해상도 파형으로부터 18가지 스위칭 성능 지표를 추출하는 핵심 모듈

4.1 전처리 및 기준 시각 탐지

이동 평균 필터링

Z채널의 4개 신호(Z_I_D, Z_V_DS, Z_V_GS, Z_I_out)에 movmean(signal, 5) 필터를 적용

정규화

$$Z_\text{ID,norm} = \frac{Z\_I\_D_f}{I_{DS}}, \qquad Z_\text{VGS,norm} = \frac{Z\_V\_GS_f - \min(Z\_V\_GS_f)}{\max(Z\_V\_GS_f) - \min(Z\_V\_GS_f)}$$

t_ton_ref — 턴온 기준 시각

Z_VGS_norm이 50% 지점을 처음으로 상향 통과하는 시점(선형 보간)

t_toff_ref — 턴오프 기준 시각

t_on_mid 이후의 Z_VGS_norm에서 50% 지점을 처음으로 하향 통과하는 시점

V_DS_ref — 실측 버스 전압

턴온 직전 500ns 구간의 Z_V_DS_f 평균. 이후 V_DS 정규화 및 에너지 임계값(5%)이 갱신됨.

$$V_{DS,\text{ref}} = \text{mean}\!\left(Z\_V\_DS_f\big|_{t \in [t_\text{ton,ref}-500\text{ns},\; t_\text{ton,ref})}\right), \qquad Z_\text{VDS,norm} = \frac{Z\_V\_DS_f}{V_{DS,\text{ref}}}$$

4.2 t_on_mid — 턴온/턴오프 분리 경계

두 번째 펄스에서 턴온과 턴오프의 분리 기준이 되는 지점을 의미(on 구간의 중간 지점). 링잉 구간에서의 오탐을 방지하기 위해 2단계 방식으로 경계를 산출

Step 1: t_ID_10r 탐지

t_ton_ref − 500ns 이후 Z_ID_norm이 처음으로 10%를 통과하는 시점.

Step 2: t_on_mid 계산

$$t_\text{on\_mid} = t_\text{ID,10r} + \bigl(t_\text{zoom,end} - t_\text{ID,10r}\bigr) \times 0.5$$

Z채널 후반부의 절반 지점으로, ON 상태 평탄 구간 내부에 위치. 이 시점 이후에서만 턴오프 관련 탐색을 수행하여 턴온 링잉 구간의 오탐을 방지.

I_DS_peak_off — 턴오프 전류 기준

t_toff_ref 직전 300ns 구간의 평균값(mean)을 사용. 이는 턴온 오버슈트가 소멸된 후 안정화된 전류를 정확하게 반영함.
$$I_{DS,\text{peak\_off}} = \text{mean}\!\left(Z\_I\_D_f\big|_{t \in [t_\text{toff,ref}-300\text{ns},\; t_\text{toff,ref})}\right)$$

4.3 E_on / E_off — 스위칭 손실

E_on

t_ton_ref 이후의 Z채널에서:

경계조건설명
t1 (시작)$$Z\_I\_D_f \geq I_{DS} \times 5\%$$전류가 설정값의 5%에 도달하는 시점
t2 (종료)$$Z\_V\_DS_f \leq V_{DS,\text{ref}} \times 5\%$$전압이 실측 버스전압의 5%까지 하강한 시점

t2에 해당하는 크로싱이 없으면(전압이 완전히 내려가지 않는 경우), Z_V_DS_f의 최소값 위치를 t2로 사용.

$$E_\text{on} = \left|\int_{t_1}^{t_2} V_{DS}(t) \cdot I_D(t)\, dt\right|$$

적분에는 원본(비필터) 신호 Z_V_DS, Z_I_D를 사용하며, MATLAB trapz (사다리꼴 적분)로 계산. 결과 단위는 [J]이며, 출력 시 [mJ]로 환산됨.

E_off

턴온과 턴오프 사이 중간 시점(t_mid)부터 탐색을 시작

$$t_\text{mid} = \frac{t_\text{ton,ref} + t_\text{toff,ref}}{2}$$
경계조건설명
t1 (시작)$$Z\_V\_DS_f \geq V_{DS,\text{ref}} \times 5\%$$전압이 버스전압의 5%까지 상승하는 시점 (t_mid 이후)
t2 (종료)$$Z\_I\_D_f \leq I_{DS} \times 5\%$$전류가 설정값의 5% 이하로 하강하는 시점
$$E_\text{off} = \left|\int_{t_1}^{t_2} V_{DS}(t) \cdot I_D(t)\, dt\right|$$

E_on과 동일하게 원본 신호 기반 사다리꼴 적분 수행

4.4 t_d_on / t_d_off — 딜레이 타임

t_d_on (턴-온 딜레이)

: 게이트 전압이 10% 상승하는 시점부터 드레인 전류가 10% 상승하는 시점까지의 시간 차이

$$t_{d,\text{on}} = t_\text{ID,10r} - t_\text{VGS,10r}$$
시점정의탐색 시작점
t_VGS_10rZ_VGS_norm이 10%를 통과t_ton_ref − 500ns
t_ID_10rZ_ID_norm이 10%를 통과 (I_DS 기준)t_ton_ref − 500ns

t_d_off (턴-오프 딜레이)

: 게이트 전압이 90%에서 하강하는 시점부터 드레인-소스 전압이 10% 상승하는 시점까지의 시간 차이

$$t_{d,\text{off}} = t_\text{VDS,10r} - t_\text{VGS,90f}$$

t_VGS_90f는 find_crossing_bwd(역방향 탐색)로 t_toff_ref + 500ns에서 뒤에서부터 탐색

시점정의탐색 시작점
t_VGS_90fZ_VGS_norm이 90%를 통과t_toff_ref + 500ns
t_VDS_10rZ_VDS_norm이 10%를 통과 (V_DS_ref 기준)t_ton_ref − 500ns

4.5 전류 및 전압 Rise / Fall 타임

기준값: 턴온 전류 기준 = I_DS(설정값), 턴오프 전류 기준 = I_DS_peak_off(실측 평균). 전압 기준 = V_DS_ref(실측 버스 전압).

t_r_I — 턴온 전류 상승 시간

I_D가 I_DS의 10%에서 90%까지 상승하는 데 걸리는 시간

$$t_{r,I} = t_\text{ID,90r} - t_\text{ID,10r} \quad \text{(탐색 구간: [t\_ID\_10r, t\_on\_mid])}$$

* t_ID_10r은 t_ton_ref - 500ns 이후 순방향 탐색

* t_ID_90r는 t_ID_10r 이후 순방향 탐색

t_f_V — 턴온 전압 하강 시간

V_DS가 V_DS_ref의 90%에서 10%까지 하강하는 데 걸리는 시간
$$t_{f,V} = t_\text{VDS,10f} - t_\text{VDS,90f}$$

* t_VDS_90f은 t_ton_ref - 500ns 이후 순방향 탐색

* t_VDS_10f는 t_VDS_90f 이후 순방향 탐색

t_r_V — 턴오프 전압 상승 시간

V_DS가 V_DS_ref의 10%에서 90%까지 상승하는 데 걸리는 시간

$$t_{r,V} = t_\text{VDS,90r} - t_\text{VDS,10r}$$

* t_VDS_10r은 t_toff_ref - 500ns 이후 순방향 탐색

* t_VDS_90r는 t_VDS_10r 이후 순방향 탐색

t_f_I — 턴오프 전류 하강 시간

I_D가 I_DS_peak_off의 90%에서 10%까지 하강하는 데 걸리는 시간

$$t_{f,I} = t_\text{ID,10f} - t_\text{ID,90f}$$

* t_ID_90f는 데이터의 뒷 부분부터 역방향으로 탐색(find_crossing_bwd)하여 턴오프 이전 데이터와 관계 없이 실제 하강 시작점을 정확히 포착

* t_ID_10f는 t_ID_90f 이후 순방향 탐색

$$Z_\text{ID,off,norm} = \frac{Z\_I\_D_f}{I_{DS,\text{peak\_off}}}$$

4.6 I_peak_on / V_peak_off — 피크값

I_peak_on — 턴온 전류 오버슈트

t_ton_ref 이후 2μs 이내 구간에서 Z_I_D_f의 최대값. 역회복 전류(reverse recovery current)에 의한 전류 스파이크를 포착

$$I_\text{peak,on} = \max\!\left(Z\_I\_D_f\big|_{t \in [t_\text{ton,ref},\; t_\text{ton,ref}+2\mu s]}\right)$$

V_peak_off — 턴오프 전압 오버슈트

t_toff_ref 이후 2μs 이내 구간에서 Z_V_DS_f의 최대값. 기생 인덕턴스에 의한 전압 스파이크를 포착

$$V_\text{peak,off} = \max\!\left(Z\_V\_DS_f\big|_{t \in [t_\text{toff,ref},\; t_\text{toff,ref}+2\mu s]}\right)$$

V_peak_off_pct — 오버슈트 비율

$$V_\text{peak,off,\%} = \left(\frac{V_\text{peak,off}}{V_{DS,\text{ref}}} - 1\right) \times 100\;\;[\%]$$

예를 들어 10%이면 버스 전압의 110%까지 전압 스파이크가 발생한 것. (측정 되지만 따로 출력하고 있지는 않음.) 변경 가능

4.7 Q_rr — 역회복 전하

턴온 시 다이오드의 역회복(reverse recovery)으로 인해 I_D가 정상 상태 전류를 초과하는 구간의 전하량

기준 전류 결정

I_peak_on 시점의 Z_I_out_f 실측값을 기준 전류(I_out_qrr)로 사용. 설정값 I_DS가 아닌 실제 출력 전류를 사용하여 정확도를 높임.

적분 구간

t_ID_90r ~ (t_ID_90r + 3μs) 구간에서 I_peak_on 이후 I_D가 I_out_qrr 이하로 하강하는 시점(t_rr_end)까지, I_D > I_out_qrr인 구간만 적분

$$Q_{rr} = \left|\int_{t_\text{ID,90r}}^{t_\text{rr,end}} \bigl(I_D(t) - I_\text{out,qrr}\bigr)\, dt\right|, \quad I_D > I_\text{out,qrr}$$

원본(비필터) 신호 Z_I_D를 사용하며, I_out_qrr는 필터된 Z_I_out_f에서 추출. 결과 단위는 [C]이고, 출력 시 μC로 환산됨.

4.8 f_ringing / ζ — 링잉 주파수 및 감쇠비

① 전류가 아닌 전압 링잉을 기준으로 지표 추출
② 탐색 시작 시점: t2_on/t2_off(에너지 적분 종료 시점)
③ 탐색 구간: t2_on/t2_off 이후 400ns 동안
④ 감쇠비(ζ) 동시 계산

f_ringing_on / f_ringing_off

지표신호구간
f_ringing_onZ_V_DS_ft2_on ~ t2_on + 400ns
f_ringing_offZ_V_DS_ft2_off ~ t2_off + 400ns

1단계: DC 성분 제거 → AC 성분 추출

$$v_\text{AC}(t) = v_f(t) - \overline{v_f}$$

2단계: findpeaks로 피크 탐지 (MinPeakProminence = 진폭 × 10%)

$$f_\text{ringing} = \frac{1}{\overline{\Delta t_\text{peaks}}}$$

ζ (zeta) — 감쇠비

대수 감쇠율(Logarithmic Decrement) 방법으로 계산. 연속 피크 진폭의 비율에서 감쇠율을 추출.

$$\delta_k = \ln\!\left(\frac{A_k}{A_{k+1}}\right), \qquad \bar{\delta} = \text{mean}(\delta_k)$$
$$\zeta = \frac{\bar{\delta}}{\sqrt{4\pi^2 + \bar{\delta}^2}}$$

0 < ζ < 1 범위 내 유효한 값만 반환하고, 범위 밖이면 NaN 처리. 양수 피크가 2개 미만이면 음수 피크(극소값 절댓값)도 함께 사용.

4.9 V_on_drop — 도통 전압 강하

디바이스가 완전히 켜진 상태에서의 드레인-소스(또는 콜렉터-이미터) 전압 강하로, 디바이스의 온 저항(R_DS(on)) 특성을 나타냄.

측정 구간

E_on 적분 종료 시점(t2_on) 이후 링잉 소멸 여유 시간(500ns)을 두고, t_on_mid(ON 상태 중앙)까지의 구간에서 Z_V_DS의 평균을 취함.

$$V_\text{on,drop} = \text{mean}\!\left(Z\_V\_DS\big|_{t \in [t_{2,\text{on}}+500\text{ns},\; t_\text{on\_mid}]}\right)$$

이 구간이 너무 짧으면(데이터 10포인트 미만), t2_on ~ t_on_mid 전체 구간을 사용

물리적 의미: $V_\text{on,drop} \approx I_{DS} \times R_{DS(\text{on})}$ 로, 도통 시 전압 강하를 의미

→ 전류값에 따라 증가하는 경향을 보여야 하지만, 현재 전압에 비례하는 특성을 보이며 전체적으로 매우 큰 값(이론값의 최대 50배)이 측정되고 있음. 검토 필요

4.10 I_out_actual — 실측 출력 전류

턴오프 직전 200ns 구간의 Z_I_out_f 평균값. Plot_LUT_Maps에서 Y축(세로축)으로 사용됨.

$$I_\text{out,actual} = \text{mean}\!\left(Z\_I\_out_f\big|_{t \in [t_\text{toff,ref}-200\text{ns},\; t_\text{toff,ref})}\right)$$

설정값 I_DS 대신 실측값을 사용하므로, 인덕터 전류 상승에 의한 설정값과의 차이를 정확히 반영함.

4.11 Crossing 탐지 함수 구조

'교차(crossing)' 방식이 아닌,'도달(threshold reach)' 방식을 채택함으로써 신호가 이미 임계값 위/아래에 있는 경우에도 정상 탐지되도록 설정
함수탐색 방향용도
find_crossing_fwd순방향 (t_after 이후 →)대부분의 크로싱 탐지
find_crossing_fwd_window순방향 + 윈도우 상한t_r_I (t_ID_90r 탐색 시 t_on_mid 상한 제한)
find_crossing_bwd역방향 (← t_before 이전)t_VGS_90f, t_ID_90f 등 하강 시작점 탐색

모든 함수는 선형 보간으로 서브-샘플 정밀도의 크로싱 시각을 계산

$$t_c = t_{i-1} + \frac{L - v_{i-1}}{v_i - v_{i-1}} \cdot (t_i - t_{i-1})$$
